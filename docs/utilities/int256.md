## Overview

`Int256` is a dependency-free signed 256-bit integer. It stores four 64-bit
limbs and uses two's-complement arithmetic. The alias `i256` names the same
type. Arithmetic and out-of-range parsing wrap modulo $2^{256}$; comparisons
and output use the signed range $[-2^{255}, 2^{255}-1]$.

Use [`Int512`](int512.md) or [`Int1024`](int1024.md) when an exact intermediate
needs more bits, and [`BigInt`](bigint.md) when the required width is unbounded.

## Interface

| Method / operator | Description | Time | Memory |
| --- | --- | --- | --- |
| `Int256()` | Constructs zero. | $O(1)$ | $O(1)$ |
| `Int256(Integer value)` | Sign-extends or zero-extends an integral type of at most 64 bits. | $O(1)$ | $O(1)$ |
| `explicit Int256(std::string_view text)` | Parses signed decimal text. | $O(1)$ | $O(1)$ |
| `operator=(text)`, `read(text)` | Replaces the value with parsed decimal text. | $O(1)$ | $O(1)$ |
| `is_zero()`, `is_negative()`, `sign()` | Queries whether the value is zero or negative, or returns `-1`, `0`, or `1`. | $O(1)$ | $O(1)$ |
| unary `+`, unary `-` | Returns the value or its two's-complement negation. | $O(1)$ | $O(1)$ |
| `+`, `-`, `+=`, `-=` | Adds or subtracts modulo $2^{256}$. | $O(1)$ | $O(1)$ |
| `*`, `*=` | Multiplies modulo $2^{256}$. | $O(1)$ | $O(1)$ |
| `/`, `/=`, `%`, `%=` | Performs signed division and remainder. | $O(1)$ | $O(1)$ |
| `divmod(a, b)` | Returns the quotient and remainder together. | $O(1)$ | $O(1)$ |
| comparisons | Compares signed values. | $O(1)$ | $O(1)$ |
| `to_string()`, `parse_int256(text)` | Formats or parses decimal text. | $O(1)$ | $O(1)$ |
| `operator<<`, `operator>>` | Writes or reads signed decimal text. | $O(1)$ | $O(1)$ |

The bounds are constant for the fixed four-limb representation. Multiplication
uses 16 limb products, binary long division performs 256 iterations, and
the decimal magnitude has at most 77 digits.

Division truncates toward zero; the remainder follows the dividend's sign.
Division by zero throws `std::domain_error`. Parsing accepts one optional sign
and throws `std::invalid_argument` for malformed text. The quotient
$-2^{255}/-1$ wraps to $-2^{255}$.

## Example

```cpp
#include "utilities/int256.hpp"

#include <iostream>

int main() {
    using m1une::utilities::Int256;

    Int256 value("10000000000000000000000000000000000000000");
    auto [quotient, remainder] = divmod(value, Int256(97));
    std::cout << quotient << " " << remainder << "\n";
}
```
