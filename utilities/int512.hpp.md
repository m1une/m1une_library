## Overview

`Int512` is a dependency-free signed 512-bit integer for exact predicates and
other calculations that need several times the width of `long long` but do not
need a general arbitrary-precision integer. It stores eight little-endian
64-bit limbs and uses two's-complement arithmetic. The related
[`Int256`](int256.md) and [`Int1024`](int1024.md) types have the same API.

The alias `i512` names the same type. Arithmetic is performed modulo $2^{512}$,
and comparisons and decimal output interpret the result as a signed value in
$[-2^{511}, 2^{511}-1]$.

Use [`BigInt`](bigint.md) instead when division or unbounded precision is
required.

The stated bounds are constant because the width is fixed. In terms of the
eight limbs, addition takes linear time, multiplication takes quadratic time,
and binary long division takes $512$ iterations with linear-time limb work.
Parsing and formatting are linear in the number of digits times the limb count;
the decimal magnitude has at most 154 digits (155 characters including a
possible minus sign).

Division truncates toward zero. A nonzero remainder has the dividend's sign and
smaller magnitude than the divisor. Division by zero throws `std::domain_error`.
The exceptional mathematical quotient $-2^{511}/-1$ wraps to $-2^{511}$.

Parsing rejects empty, sign-only, and nondigit text with
`std::invalid_argument`. Values outside the signed range wrap modulo $2^{512}$,
matching arithmetic. Bit operations are intentionally not provided.
